# 时区边界测试清单

## 概述

本清单涵盖时区相关的各类边界场景测试，确保跨时区家族拜年排期的正确性。

## 一、基本时区转换测试

### 1.1 多时区显示正确性
- [ ] **中美时差验证**: 北京时间 20:00 对应洛杉矶时间 04:00 (冬令时) / 05:00 (夏令时)
- [ ] **美东美西差异**: 纽约时间比洛杉矶时间早 3 小时
- [ ] **北京时间基准**: 所有时段创建默认以北京时间为准
- [ ] **切换用户时区**: 切换不同时区的用户，日历显示正确转换

### 1.2 日期变更线处理
- [ ] **跨天显示**: 北京的 2月17日 20:00 → 洛杉矶是 2月17日 04:00 (同一天)
- [ ] **凌晨时段**: 北京的 2月18日 02:00 → 洛杉矶是 2月17日 10:00 (前一天)
- [ ] **跨日时段**: 一个时段跨越当地时间的午夜，正确显示为两天

## 二、夏令时 (DST) 边界测试

### 2.1 美国夏令时 (三月第二个周日)
- [ ] **Spring Forward**: 3月第二个周日凌晨 2:00 直接跳到 3:00，缺失的 1 小时
  - [ ] 可用性窗口包含此时间段时的处理
  - [ ] 预约时段落在缺失时间的处理
- [ ] **DST 当天时长**: 当天只有 23 小时
- [ ] **DST 前后偏移变化**: 冬令时 PST (UTC-8) → 夏令时 PDT (UTC-7)

### 2.2 美国冬令时 (十一月第一个周日)
- [ ] **Fall Back**: 11月第一个周日凌晨 2:00 变回 1:00，重复的 1 小时
  - [ ] 可用性窗口包含此时间段时的处理
  - [ ] 1:00-2:00 这小时出现两次，如何区分
- [ ] **DST 当天时长**: 当天有 25 小时
- [ ] **DST 前后偏移变化**: 夏令时 PDT (UTC-7) → 冬令时 PST (UTC-8)

### 2.3 中国时区特殊情况
- [ ] **中国无夏令时**: 亚洲/上海时区全年 UTC+8
- [ ] **跨 DST 边界对比**: 同一天中美两国，一边有 DST 变化一边没有

## 三、跨日时段切片测试

### 3.1 单时区跨日
- [ ] **晚间到凌晨**: 22:00 - 次日 02:00，正确切分为两天
- [ ] **整天覆盖**: 00:00 - 23:59，作为一天处理
- [ ] **跨越多天**: 持续 48 小时的时段，正确切分每天的片段

### 3.2 多时区跨日组合
- [ ] **北京白天 vs 美国夜间**: 北京 10:00-12:00 = 洛杉矶 18:00-20:00 (前一天)
- [ ] **北京晚间 vs 美国早晨**: 北京 20:00-22:00 = 洛杉矶 04:00-06:00 (同一天)
- [ ] **多时区日历对齐**: 周视图中，不同时区的时段正确对应到各自日期

### 3.3 可用性窗口跨日
- [ ] **成员标记跨日窗口**: 比如 20:00 - 次日 02:00
- [ ] **跨日窗口参与重叠计算**: 正确计算 UTC 时间区间的重叠

## 四、重叠空档算法边界测试

### 4.1 成员数量边界
- [ ] **单人情况**: 只有一个成员标记时间，不生成推荐 (至少 2 人)
- [ ] **两人情况**: 两个成员的重叠，正确计算
- [ ] **全员重叠**: 所有成员都有空的时间段
- [ ] **部分重叠**: 只有部分成员有空的时间段

### 4.2 时间粒度测试
- [ ] **30 分钟粒度**: 默认粒度，验证准确性
- [ ] **15 分钟粒度**: 更细粒度，验证性能与精度
- [ ] **1 小时粒度**: 更粗粒度，验证合并逻辑

### 4.3 边界重叠情况
- [ ] **刚好相接**: A 的结束时间 = B 的开始时间，算重叠吗？
- [ ] **部分重叠**: 只有部分时间重叠
- [ ] **完全包含**: 一个时段完全包含在另一个时段内
- [ ] **完全不重叠**: 两个时段没有交集

### 4.4 推荐算法边界
- [ ] **少于 3 个推荐**: 可用时段不足 3 个时的处理
- [ ] **评分相同**: 两个时段评分相同时的排序
- [ ] **长辈优先**: 包含长辈的时段是否正确加权
- [ ] **深夜惩罚**: 当地时间 22:00-07:00 的时段评分降低

## 五、预约与排队边界测试

### 5.1 满员与候补
- [ ] **刚好满员**: 最后一个名额被预约
- [ ] **超员候补**: 满员后再预约，进入候补队列
- [ ] **候补排序**: 按预约时间先后排序候补
- [ ] **取消递补**: 有人取消，候补队首自动递补
- [ ] **候补取消**: 候补用户取消，后面的人顺位前移

### 5.2 重复预约
- [ ] **不能重复预约**: 同一用户不能预约同一时段两次
- [ ] **候补转确认**: 候补用户递补后，状态正确变更
- [ ] **取消后重约**: 取消后可以重新预约

### 5.3 权限边界
- [ ] **创建时段**: 仅管理员可创建
- [ ] **私密备注**: 仅长辈可见，其他人看不到该字段
- [ ] **修改时段**: 仅管理员可修改/删除

## 八、性能边界测试

### 8.1 大量数据
- [ ] **50+ 成员**: 大量成员的重叠计算性能

## 测试工具建议

4. 手动切换系统时区测试 UI 显示
